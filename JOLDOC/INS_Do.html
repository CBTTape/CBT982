DO - Execute and Immediate Instruction

Purpose

 DO  and END Statements are used to construct 'DO blocks. A DO block begins with a DO Statement and ends with a matching END Statement.

        

 DO ;
jol statement...
jol statement...

END ;

Do End Blocks


A DO block is usually used after an IF statement and signifies the beginning of a group of instructions which may then be executed or bypassed subject to the result of an IF.  A simple DO block begins with a simple DO statement and has the form

DO;
statement-1;
statement-2;
statement-3;
END;


The following is an example:

Do;
Copy INPUT To OUTPUT;
Catalog OUTPUT;
End;


A simple DO block may be regarded as a simple Jol statement, and may appear anywhere in the program that a single executable statement may appear.  Each statement within the simple DO block may be any Jol statement, including both executable and declarative statements.  The executable statements (if any) within the DO block are executed in normal sequence just as if they were not enclosed in the DO and END statements, unless they, too, are executed subject to other IF statements.

Nesting
 Blocks may be nested within each other as shown in the following:

DO;
statement-1;
statement-2;
DO;
statement-a;
statement-b;
statement-c;
END;
statement-3;
statement-4;
END;

The first DO statement and the second END statement bracket one simple DO block.  The second DO statement and the first END statement bracket a different DO block inside the first one.
Indentation Notice how indentation (using spaces) can be used to make the sequence readable; it can be seen at a glance that one DO block is nested inside another.  This practice should be followed when writing Jol programs.

Notes
  1. The maximum number of levels of DOs allowed is eight (8).
  2. A DO group is terminated by an END statement.
  3. A DO group may be repeated with the GOTO or REDO instructions.

Examples

 Example 1.

1     IF %RESTART=1
2     THEN DO;
3          DCL INPUT DATA SET GEN.DATA(0);
4          DISPLAY 'RESTARTING AT STEP 10';
5          STARTAT STEP10;
6     END;
7     other instructions;

Statement 1    tests the symbolic variable %RESTART, and if it is one (1), then statement 2 is executed.  If it is not true, that is %RESTART is not 1, then statements 2 to 6 are ignored and instructions at statement 7 are executed.

Statement 2    is the DO instruction.  Normally, only the next instruction after an IF is executed, but when the DO END instructions are used, a group of instructions may be executed or interpreted as if they were only one statement.

Statement 3    declares INPUT to be a data set called GEN.DATA(0).

Statement 4    will display 'RESTARTING AT STEP 10' when the job starts.

Statement 5    issues the STARTAT instruction.  All steps until STEP 10 will be ignored and no RUNs, CATALOGs etc will be executed.

Statement 6    is the END for the group of instructions commenced by the DO at statement 2.


Example 2.
     
STEP2:      RUN VALIDATE;

IF STEP2=0
THEN DO;
RUN UPDATE;
IF UPDATE=0
THEN RUN REPORT;
END;
IF MAXCC=0
THEN CATALOG NEWMAST;
ELSE STOP 'ERROR OCCURRED - FIX AND RESUBMIT';

The VALIDATE program is RUN.

If it returns a zero then the IF statement (IF STEP2=0) will be true, and the next group of instructions will be executed.  The DO group includes running the UPDATE and REPORTprograms, and another IF test.

Following the END statement for the DO group is a test for MAXCC.  This means that if all the programs returned zero, the CATALOG NEWMAST instruction will execute.

If all the programs did not return zero, the STOP instruction will execute and terminate the job with the message.
