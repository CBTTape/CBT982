LINK - Execute Command

Purpose

 The LINK Command invokes the system Linkage Editor to convert the object text from the COB, PL1, FORTRAN or ASM Commands to load module format.  The Load module can then be executed with the RUN, EXEC or similar commands

 

LINK          { data-set-list     }   options ;
or      { dsid-list    }

where options are:

LOAD       dsid | data-set-name  

NAME (program-name)
  
LIB         { data-set-name [|| data-set-name]    }
or  { dsid [|| dsid ... ]    }

{  PRINT        data-set-name | dsid  | NOPRINT     }
{  SYSOUT      * | class    }

[  PLILIB        ]
[  PLICMIX     ]
[  PLIBASE    ]
[  FORTLIB    ]
[  COBLIB      ]

INCLUDE        {  data-set-name   [(member-list)]    }
      { dsid    }

AC=0 | 1
SSI    =     system status information
SIZE     =     virtual storage size
DCBS     =     blocksize
LIST | NOLIST
MAP | NOMAP
CROSSREF | XREF
OVERLAY | OVLY | NOVLY
TEST | NOTEST
LET | NOLET
RENT | NORENT
REFR | NOREFR
NCAL | NONCAL
XCAL | NOXCAL
OL | NOOL
PANEL | NOPANEL
NE | NONE
DC | NODC
SCTR | NOSCTR
      


Notes
  1. Default values are satisfactory for most purposes.
  2. LOADGO may be a faster alternative to LINK.
  3. If no name is specified in the LOAD parameter, TEMPNAME is the default.
  4. SSI defaults to the current date and time.  Thus day 156 in 2009, time 14:15 will become SSI 91561415.  The SSI will be printed when the program RUNs, thus providing an audit trail of which version of a production program ran.
  5. If PRINT is not specified the printer output is placed in a SYSOUT data set, which defaults to the TSO sysout class. This can be examined with the OUT Command.
  6. If LOAD is not specified, a temporary data set &LOAD is created and the module stored under the name TEMPNAME.

Examples

 1        LINK OBJ LOAD('PROD.LOAD(PAYUPDTE)') COBLIB;

The object code specified will be linked and stored in the data set PROD.LOAD as member PAYUPDTE.
