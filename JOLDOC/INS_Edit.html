EDIT - Immediate Command

Purpose

 The EDIT Instruction splits information contained in one symbolic variable according to specifications in a format list.  The results are placed in other specified symbolic variables.  Multiple SUBSTR functions may be used to perform the same function.

        

EDIT      input-symbolic-name
      [ ( ] output-symbolic-name-list [ ) ]

     [ ( ]    A(n)   [,...] [ ) ]       ;
              X(n)

Editing Symbolic Variable Data


The EDIT instruction is a very convenient method by which data in a symbolic variable can be transferred to other variables under control of a format item.  Multiple SUBSTR functions could perform the same operations, but the EDIT is a convenient method when many items are required.

NB    The EDIT instruction is interpreted by the Jol preprocessor.  This means that only symbolic variables can be altered.  You cannot EDIT symbolic variables at Execution Time.

Notes
  1. Names in the EDIT should not have the % symbol preceding them.  Jol will replace any names coded with % with the current contents of the symbolic variables.
This allows you to use parameters on Macro commands as names in the EDIT.  For example:

%LIST='A,B,C A(2)';
EDIT SYMNAME %LIST;

After replacement of the Symbolic Parameters this results in:

EDIT SYMNAME A,B,C A(2)

which will result in %A, %B, %C having the EDITed information.

  1. Format item A specifies that n characters from the input string are to be copied into the current output location. For example:
%X = 1234;
EDIT X NUM1,NUM2 A(2);

After the EDIT, %NUM1 will contain 12 and %NUM2 will contain 34.

  1. Format item X specifies that n characters are to be skipped on the input string.  For example:

%X = 'ABCDEFGH';
EDIT X CHAR1,CHAR2 A(2),X(2);

After the EDIT, %CHAR1 will contain AB, characters CD will have been skipped as specified by the X(2), %CHAR2 will contain EF, and the last characters (GH) will also be skipped.

  1. If there are more names in the output list than format items, the format items are reused until either the input string or the output name list is exhausted.  For example:
EDIT X C,D A(2);

The information in %X is copied to %C and %D, 2 bytes at a time, as specified by the one format item A(2).  The A(2) format is used twice because there are two output location names specified.

  1. If the output variable names are not declared, they will be implicitly declared. For example:
EDIT Y E,F A(5);

If the %E and %F were not declared prior to the EDIT instruction, they will be declared and initialized to null (") by the EDIT.  Information will then be copied from %Y to %E and %F.

Examples

 Examples of the EDIT instruction

1        %INPUT = '123456789'     /* Initialize %INPUT */
EDIT INPUT (A,B,C) (A(3));

After the EDIT A will contain '123', B will contain '456', and C will contain '789'.

2        %INPUT = '123456789'     /* Initialize %INPUT */
EDIT INPUT A,B,C X(2), A(2);

After the EDIT A will contain '34', B will contain '78', and C will contain ''(null).

3        EDIT DAY DAY A(3);

Suppose Jol was running on Monday, then %DAY is automatically set to 'MONDAY'.  The EDIT instruction above will set %DAY to 'MON'.

4        GET INFILE01;

%PARM = 'CARD(1)';

EDIT PARM
S10PARM,
S11PARM,
SIZE
A(8);

IF %SIZE < 100 THEN STOP
'SIZE OF %SIZE INVALID: MUST BE > 100';

The instructions above read data from a card image file and then EDIT the information obtained into other symbolic variables.

If the card image read contained:

ABCDEFGH,12345678,00000020

then:     %S10PARM will contain ABCDEFGH

and:      %S11PARM will contain 12345678

and:      %SIZE will contain 00000020 (or 20 when used arithmetically).

The IF test for %SIZE will be true, and the STOP instruction will be interpreted and the job not executed.
