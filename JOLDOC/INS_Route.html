ROUTE - Execute Command

Purpose

 The LIST Command will print all, or part of, a data set in character, and/or hexadecimal format.  Record numbers will be printed with each record, and each set of eight characters will be separated from the next by two blanks.

 

ROUTE   { PRINT  | PUNCH }

        [TO]   

        { LOCAL  |  RMTn   |  PRINTERn  |  PUNCHn  }
      


Routing Output to Different Locations

The ROUTE command is used to alter the default routing of your job's output.  If you submit work from a remote terminal your printed results will normally be returned to that terminal; the ROUTE command allows you to alter the location at which the printing or punching will occur.

Similarly, you may be submitting work from the Central location, and wish it to be printed at a Remote Terminal.  Again, the ROUTE command will facilitate this.

Your installation manager will be able to tell you the names and locations of any Remote Terminals you have.

For VS users, the DEST parameter on the PRINTer definition may be used to direct part of your job's output to a Remote Terminal or Centrally, as the case may be.

Notes
  1. This command creates a ROUTE card for HASP or JES2, and a FORMAT card for ASP or JES3.
  2. The names RMTn, PRINTERn, PRINTERn and PUNCHn are installation defined.

Examples
  1. ROUTE PRINT TO LOCAL;
The ROUTE command instructs the Operating System to print all the results LOCALly.

  1. ROUTE PRINT TO RMT3;
The ROUTE command directs the Operating System to print all the results at the destination known to it as RMT3.
