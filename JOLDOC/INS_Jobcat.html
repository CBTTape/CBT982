JOBCAT - Execute Command

Purpose

 The JOBCAT Statement defines a private catalog to be made available by the system to an entire job.

 

JOBCAT    dataset1 [. . . dataset2]    

       

Private Catalogs

Unless you inform the system that any data set you will use can be found in a private or temporary catalog, the system expects to find it in the system catalog.

One way to tell the system that the data set can be found in a private catalog is to use the JOBCAT Command to inform the system that the named catalog is to be searched for all data sets in the job before the system catalog is searched.  When you use the JOBCAT command, each time you request a data set, the system looks for it in the specified catalog.  Another way is to use the STEPCAT option in the Program declaration.

Notes
  1. The JOBCAT Command may be coded anywhere in a Jol Program.
  2. The maximum number of catalogs that may be specified on the command is five.
  3. If you include a JOBCAT command in your Jol program, each time the job requests a data set, the system first searches the private catalog.  If it does not find the data set there, it next searches the system catalog.
  4. Use a STEPCAT statement, described under the program Declaration, to define a private catalog to be made available to one job step in a job.
  5. If you include a STEPCAT statement for a program and a JOBCAT command for the entire job, the system first searches the step catalog and then the system catalog for the requested program. The job catalog is ignored for that step.

Examples
Examples of the JOBCAT Command.

1        JOBCAT PRIVATE.LIBRARY;

The library known as PRIVATE.LIBRARY is defined as the Job Catalog.

2        JOBCAT PRIVATE.CATALOG,
SECOND.CATALOG;
