SHOWDBF  - Immediate Instruction

Purpose

 The SHOWDFB instruction retrieves information from the Jol Data Base of Data Sets.

The SHOWDBF instruction allows Users to scroll through the data base, and select a data set.  The name selected can then be used in other Jol commands.



 

SHOWDBF        [ optional parameters ]

where 'optional parameters' are:
LEVEL ( high-level-index ) Limits the data to be displayed.  Starts the display at the index specified.

The default is the userid index.
MODELS Specifies that MODELS only are to be displayed.
PRINTER Specifies that PRINTERS only are to be displayed.
ALL Specifies that MODELS, PRINTERS and Data Sets are to be displayed.



The Jol Data Set Data Base

The Jol data base of data sets contains attributes of data sets.  This data base is accessed automatically when new data sets are created, and such information as record-format, space and volume information from the data base is automatically merged with new data set information.

This allows data set information to be altered once, and when jobs are executed, the latest data set attributes are used automatically.

Finding Data Set Information

 Often, users and Jol macros need to find information about data sets.  The SHOWDBF and DBFFIND instructions (when used in combination) access the data base, and returns information in symbolic variables.

When the data set in found, the information is returned in the following symbolic variables:


   LASTCC               0 if data set is selected, else non-zero

   SYSDSN               Data Set Name, PRINTER or
MODEL selected, or null.

Notes
  1. See the DBFFIND instruction for extracting information from the data base.

Examples
  1. SHOWDBF LEVEL('PAYROLL');
The SHOWDBF displays information from the data base on the Users Terminal, allowing him or her to select a data set.

Because LEVEL is specified, only data sets starting with PAYROLL  are displayed.
