GETTIME - Immediate Instruction

Purpose

 GETTIME resets the day, date and time Symbolic Variables.  This can be useful if Jol has been in a long wait, and you wish to find the current date and time.


GETTIME      ;
Automatically Initialized Symbolic Variables

As described in Declaring Symbolic Variables elsewhere, Jol initializes certain symbolic variables at startup time.  They are initialized once and can then be used with the Jol logic statements to control events and functions.  These variables are shown below:

%SYSUID: the System User Identification (SYSUID)
%SYSDATE: the current date, for example 09200
%SYSPREF
 the Dataset prefix or Current Directory
%SYSPFK
Program function key number from PANEL
%SYSMENU
Last Menu Selection from PANEL
%SYSPOP
Last Popup Selection from PANEL


%DAY: MONDAY, TUESDAY etc
%MONTH: JANUARY, FEBRUARY etc
%MONTHNO: 01, 02 through to 12
%DAYNO: 01 - 31
%YEAR: 2009. 2010, ...
%HOURS: 0 - 23
%MINS: 0 - 59
%SECS: 0 - 59


%SYSTEM: MVS, F4, DOS, X8, VS1 or VS2
%SPOOL: HASP, ASP, JES1, JES2, JES3 or blank
%TSOCLASS
 Contains the SYSOUT class used by TSO to retrieve output from a background job.

The GETTIME instruction resets %SYSDATE, %MONTH, %MONTHNO, %DAY, %DAYNO, and %TIME to the current date and time.

Notes
  1. The Jol compiler usually starts converting a job to an executable program, and stops after it has performed this function.  WAITs can be introduced by calling a program that waits for a specific event to occur, or by using such instructions as PANEL or READ.

Examples
  1. GETTIME ;

GETTIME resets the Symbolic Variables as described above.
