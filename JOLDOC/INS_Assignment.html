ASSIGNMENT or SET - Immediate Instruction

Purpose

 The SET or ASSIGNMENT instruction is used to change or set the value of a Symbolic Variable.

        

[%]name = {expression | function-expression } }
or
SET name  = {expression | function-expression } }

   where expression is:

[  +    ]
{ symbolic-variable }    [  -    { symbolic-variable    }    ]
{ constant }                     [  *    { constant    }    ]
{ 'character-constant' }  [ /     {'character-constant'    }    ]
 [  | |    ]

  and function-expression is one of:

CAPS (string-expression)
CENTER (string-expression, length)
CENTRE (string-expression, length)
CHAR (numeric-expression)
EOF (file-name)
INDEX (string-expression,'search string')
LEFT (string-expression,length)
LENGTH (string-expression)
LOWER (string-expression)
POS ('search string',string-expression)
RIGHT (string-expression,length)
STMT () returns current-statement-number
SUBSTR (string-expression,start-position[,length])
TYPE (string-expression)

returns 'LIT', 'CHAR', or NUM
UPPER (string-expression)
USERID () returns USERID or Jobname
WHEREX () returns current cursor column
WHEREY () returns current cursor row





Symbolic Variables and Text Replacement


A symbolic variable is a special variable; it is recognizable by a % in front of the name.  Whenever a symbolic variable name is found in instructions other than SET, Assign or DECLARE symbolic statements the current contents of the symbolic variable replaces the name.  For example:

%LOCATION = 'CHICAGO';

other instructions . . .

PRINT %LOCATION.PAYROLL.FILE;


is equivalent to:

PRINT CHICAGO.PAYROLL.FILE;



The symbolic variable %LOCATION is set to contain CHICAGO.

Later, the PRINT command prints the data set %LOCATION.PAYROLL.FILE.  Because %LOCATION contains the characters 'CHICAGO', whenever %LOCATION appears, it is replaced by CHICAGO.  Thus the PRINT becomes PRINT CHICAGO.PAYROLL.FILE.

If %LOCATION contained LONDON then the data set to be printed would have been LONDON.PAYROLL.FILE.

Thus symbolic substitution causes the symbolic variable names to be replaced by real values (numeric or character data).
Adding Flexibilty
 Symbolic variables add flexibility to command procedures by symbolizing real values; these real values can change dynamically during the command execution.  This allows command procedures using symbolic variables to execute multiple functions, or the same functions with different values.

Symbolic variables may be tested with the IF instruction or altered with the Assign or SET instructions.

The Compile Time Facilities chapter describes the facilities available in detail.
Assigning Values to Symbolics The symbolic variable assignment statement allows you to place values (numbers or character strings) into symbolic variables.  Also, the functions allow you to store their results in variables that you can then use.

You may assign contents of other symbolic variables, for example:


%A=10;                      /* Set A to 10 */

%A='CHARACTER STRING';      /* Set A to characters */

 are examples of assigning constants, and

%A=%B;                    /* Set A to contents of B */

%A='%B' || '%C';     /* Set A to contents of B appended with contents of C */

 are examples of assigning variables, and

%A=%B+10;              /* Set A to contents of B plus 10 */

 is an example of the mixture of variables and constants being used in an assignment.

 In all the above cases, the symbolic variable A receives the values on the right hand side of the equal sign.

Arithmetic Functions
You may add, subtract, multiply or divide a symbolic variable or constant with another symbolic variable or constant.


+        means add the two quantities together

-         means subtract the second quantity from the first quantity

/         means divide the first quantity by the second

*         means multiply the two quantities together


The result is then stored in the variable named on the left hand side of the Assignment or SET instruction.  Thus:

%A=10+5;             results in %A being set to 15,

and

SET X =%B-10;   results in X being set to the contents of B minus 10.
Concatenation
 The concatenate function allows you to append one string on to another.  If %A contains 'ABC' and the numerals 02 are to be appended to the string, you could code:

%A='%A' || 02;       which results in %A    containing 'ABC02'.

Note:    Any coded apostrophes are temporarily removed before the concatenation is performed, then they are replaced.  Even if only one operand had quotes, the result will have quotes.
Use of the SET Instruction
 The SET Instruction differs from the Assignment Statement in only one aspect: you can specify the name receiving the values as a symbolic name.

Suppose that %X had a name of a symbolic variable, and a 10 had to be stored in that name.  Coding %X=10;  will not do this, because this assigns a 10 to %X, thus destroying the contents of %X.  In fact, the name contained in %X must be used in the Assignment Statement.  This can be done with:

SET %X = 10;

The %X is replaced with the name (say NUMBER) and this results in

SET NUMBER=10;

which assigns the value 10 to %NUMBER.
Use of Apostrophes in the Assignment
Statement

 Any string of characters containing blanks, or special characters (commas, periods and so forth) must be enclosed in apostrophes before being assigned to the variable.  This also includes any symbolic variables that may contain strings with blanks and special characters.  This is because Jol looks at the second token of an assignment statement to see if arithmetic operations must be performed.  If there is a second token, and it is not +, -, /, * or || then an error is signalled.    Thus, to assign the string:

31 QUEEN ST, MELBOURNE

to %X, it must be in apostrophes:

%X='31 QUEEN ST, MELBOURNE';

If %X is now assigned to %Y, it must again be enclosed in apostrophes because when the %X is replaced by its contents, it will be an invalid character string.

Thus %Y=%X would be invalid, but %Y='%X' would be valid.

If %X contained the string '5 + 3' then

%Y=%X;

will result in (after replacement)

%Y=5+3; which will assign 8 to %Y.

However, %Y ='%X'; will result in '5+3' being assigned to %Y because the apostrophes signify that a direct character assignment is to be performed.  It is good practice to always place apostrophes around any character strings when assigning them.
Replacement with Quoted Strings
 Whenever a symbolic parameter name is replaced by its actual contents, any quotes are removed, and any double quotes converted to one apostrophe unless the replacement is made into a quoted string.

For example:      %DSN ='JOL.INCLUDE';
. . .
DCL INPUT DS %DSN;
results in:              DCL INPUT DS JOL.INCLUDE;


and             %MESSAGE ='THAT''S IT EVERYONE';
. . .
DISPLAY '%MESSAGE';

results in:    DISPLAY 'THAT''S IT EVERYONE';


Note: Had the DISPLAY been:

DISPLAY %MESSAGE;

then the result would have been

DISPLAY THAT'S IT EVERYONE;

That is, the two apostrophes will have been converted to one apostrophe.
Functions
 Functions return values to a variable.  Jol supports the following functions:

CAPS         (string-expression)

CAPS converts a string expression to upper case.

For example, A=CAPS('lower case string');


CENTER        (string-expression, length)
CENTRE        (string-expression, length)

CENTRE or CENTER    returns a string of length with string centered in it, with blanks added at either end.

For example, A=CENTRE('Center me',78);


CHAR        (numeric-expression)

CHAR returns the EBCDIC or ASCII representation of the number.

For example, A=CHAR(217);  set A to the character 'R'.


EOF            determines if a file that is OPENed for INPUT is at the end of file, that is if all the records have been read from the file.

EOF    returns 0 if there are still records to be read, otherwise a 1 is returned.

Example

ALLOC F(INDATA) 'SYS1.PROCLIB(Z)' SHR;
OPEN F(INDATA) INPUT;      /* open the file */

%TEST = EOF(INDATA);       /* Test EOF */
IF %TEST=0               /* Check End-of-file */
THEN GETFILE INDATA;          /* Not End-of-file, get another record */


INDEX        (string-expression,search-string)

INDEX searches a string in a symbolic variable for the occurrence of a character, or a set of characters.

The result is a 0 if the character(s) is not in the specified string, or a number which is the position in the string where the first occurrence of the character(s) was found.  This number may then be used in a SUBSTR to extract the string.

Examples

%A='ABCDEFG';               /* Set symbolic A to ABCDEFG */

%B=INDEX(A,'F');              /* %B now contains 6, this being the position of the F in the string */

%C=INDEX(A, 'AB2G');     /* %C contains zero because the characters AB2G were not in the character string ABCDEFG */


LEFT          (string-expression, length)

LEFT returns a string of length which contains the left-most length characters of string.

For example, A=LEFT('abcdefg',2) sets A to 'ab'.


LENGTH    returns the length of the contents of a symbolic variable, that is the number of characters currently assigned to it.

Examples

%A=10;
%B= LENGTH(A);          /* will set B to 2, because there are 2 digits in the characters 10 */


%A='FRIDAY 10TH MAY';
%B=LENGTH(A);          /* %B will now contain 15.  Note that the quotes are not counted, nor is it necessary to ever consider them */


LOWER     (string-expression)

LOWER converts a string expression to lower case.

For example, A=LOWER('LOWER CASE STRING');


POS            (string-string,search-expression)

POS searches a string in a symbolic variable for the occurrence of a character, or a set of characters.

The result is a 0 if the character(s) is not in the specified string, or a number which is the position in the string where the first occurrence of the character(s) was found.

See the INDEX function for examples.


RIGHT         (string-expression, length)

RIGHT returns a string of length which contains the right-most length characters of string.

For example, A=RIGHT('abcdefg',2) sets A to 'fg'.


STMT         ( )    returns current-statement-number

STMT returns the current statement number in the program.


SUBSTR    extracts sub-strings from strings of data.

The name of the symbolic variable containing the data must be specified, as well as the start position within the data from which the sub-string is to be extracted.  If the length is not specified, the remaining data is transferred to the output symbolic variable.  If the length is specified then only the amount of data specified is transferred.  If the input string is shorter than the length specified, the output string will not be padded with blanks.

Examples:

%A = 'ABCDEFG';             /* Set symbolic A to ABCDEFG */

%B = SUBSTR(A,3);         /* %B now contains CDEFG */

%C = SUBSTR (A,3,2);     /* %C now contains CD */

%D = SUBSTR(A,15);       /* %D now contains '' null because the input string is shorter than the start position specified */


TYPE         determines the type of contents in a symbolic variable.

If the contents are numeric, TYPE returns NUM.  If the contents are in quotes, TYPE returns LIT, otherwise CHAR.

Examples

%A=10
%B=TYPE(A);               /* sets %B to NUM */


%A='FRIDAY 10TH MAY';
%B=TYPE(A);               /* sets %B to LIT */


%A = FRIDAY;
%B=TYPE(A);               /* sets %B to CHAR */


UPPER      (string-expression)

UPPER converts a string expression to upper case.

For example, A=UPPER('lower case string');


USERID    ( )

USERID returns userid or  jobname


WHEREX  ( )   returns current cursor column
WHEREY  ( )   returns current cursor row

WHEREX and WHEREY return the current cursor position when executing in interactive mode, otherwise 0.  The cursor position is only valid after executing a PANEL or POPUP instruction.


Notes
  1. Symbolic variables and parameters are fully described in the chapter Compile Time Facilities.
  2. The names may be preceded by a % symbol.  They may be from one to eight alphameric or national ($, # or @) characters.  For example:

%A
%123
%$NAME
%GEN

  1. The following names are reserved and have special meanings.  Any attempt to DECLARE  them will result in an error message; use of the Assign or SET instruction will alter the contents of the named variable to the value you specify.

The following are initialized by Jol before any statements have been read:

%SYSDATE
The current date e.g., 09225
%DAY MONDAY, TUESDAY and so on

%MONTHNO 01,02 through to 12

%HOURS 0 - 23
%MINS 0 - 59
%SECS 0 - 59
%SYSTEM MFT, MVT, VS1, VS2 or VM
%SPOOL HASP, ASP, JES1, JES2, JES3 or blank


Other Symbolic Variables are automatically initialized and changed during the compile.  They are of particular interest to those writing Jol Macros.

These, and their contents, are:-

%SYSLABEL
the label on the statement invoking the macro
%SYSMACNM
or %SYSSCMD		 the name of the macro called . . . useful when you have one macro doing several related functions and each is stored as an ALIAS. You can have this to determine which "entry point" the macro was called at.
%SYSNLIST the number of LIST (non-keyword) items found on the invoking statement.
%SYSLEVEL the current level of the macro. If a macro invokes another macro it will be set at 2; in the mainline code it is set at 0, and in the first macro invocation it will be set at 1.
%SYSSTMT this variable is set at 0001 - 9999, and is altered every time a new statement is read.



  1. If the variable is not already Declared, it is implicitly declared at the current level.
  2. When used in a Macro, and the name is not defined nor declared external to the Macro, it is implicitly declared at the same level as the Macro, and the value will not be available after the Macro terminates.
  3. The SET may be used to alter the value of a Symbolic Variable by using an existing symbolic variable contents to specify the name of the variable to be altered.  For example:

%X='ABC';
SET %X=1;

Result: %ABC will contain the 1, %X will be unaltered.

  1. The maximum length of any assignment value is 253 characters.  Assigning more than 253 characters will result in truncation.
  2. All symbolic variable contents are held in character format, even if they are numeric.
  3. When any arithmetic operations are to be performed on symbolics variables, they are first converted to decimal packed numbers.    If any errors are detected during this conversion, an error is signalled and the result is forced to zero.  The maximum numberof digits in a number is allowed to be thirteen (13) : longer numbers are treated as characters.
  4. All arithmetic is performed in integers.  Thus
%A=5/3;
results in 1 being assigned to %A.
  1.  When a name is declared in the main Jol procedure, and the same name is declared in a Macro Command, any reference to that symbolic variable will result in the second definition being utilized.  When the END statement for the macro has been executed, any reference to that variable will revert to the value of the first declaration being used.
  1. All symbolic variable contents are held in character format, even if they are numeric.  When arithmetic operations are performed on symbolic variables, they are first converted to decimal packed numbers.  If errors are detected during this conversion, an error is signalled and the result is forced to zero.  The maximum number of digits in a number is allowed to be thirteen (13) : longer numbers are treated as characters

Examples

 1        A='ABC';

The symbolic variable A is set to the characters ABC.  Whenever %A is now written (except in an assignment), the characters ABC will replace it.

2        NUM = B + 5;

The symbolic NUM is set to the current contents of B, plus 5.  If B contained a 2, then NUM would now contain 7.

3        CHAR ='%A%B%C';

The symbolic CHAR is set equal to the result of concatenating the current contents of the symbolic variables A, B and C.

4        NAME='MASTER.FILE' || %NUM;

The symbolic variable NAME is set to the characters MASTER.FILE concatenated to whatever the current contents of NUM are.  If NUM contained 02, then NAME would equal MASTER.FILE0.

5        NAME = 'SET A = B+C';
...
%NAME;

Sets A to the result of B plus C.

The %NAME statement is replaced by the current value of %NAME which, in this case, is an assignment or SET instruction.  It is then executed.  Other types of instructions may also be set up in this manner.

6     This example is a macro that splits variables into two if the length of the information contained in the symbolic variable OP is greater than 71 characters.
  1. SPLITOP: MACRO;
  2.        L = LENGTH (OP);
  3.        IF %L>72
  4.        THEN DO;
  5.                 LINE2 = SUBSTR (OP,72);
  6.                OP = SUBSTR (OP, 1,71);
  7.       END;
  1.       ELSE LINE2 ='';
  1. END;


The SPLITOP command above checks the number of characters in OP and if there are more than 72 characters it will place the 73rd and subsequent characters in LINE2, and truncate OP to 72 characters.  If there are fewer than 72 characters, LINE2 will be cleared and OP will remain unaltered.
