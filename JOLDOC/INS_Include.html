INCLUDE - Immediate Instruction

Purpose

 The INCLUDE instruction copies Jol source text from the member specified in the Jol Include Library.  The source text is compiled as though it was part of the original input to the Compiler.


INCLUDE member-name ;

or

INCLUDE  'dsname'

Including Instructions or Source Text

Typically, Jol code is stored in a library called the INCLUDE library. The INCLUDE instruction can instruct Jol to read the statements from the library.

Once a procedure has been tested and debugged, you can store it in a library called the INCLUDE library.  The INCLUDE instruction may then be used to instruct Jol to read the statements from the library, instead of reading cards.  New procedures may be stored in the library with the STORPROC command and updated with the UPDATE command, or SPF/TSO or similar may be used.

Using the INCLUDE instruction to include Jol source instructions or text avoids:

·     Writing the same often-used code more than once.

·     Keypunching and handling the punched cards for that code.

·     Errors due to accidental dropping and mis-sequencing of the cards.

The library in which you store your Jol source statements is called the INCLUDE library (or libraries).  The library contains individually named members; these can contain any allowable Jol statements, even card image files (DCL name *).

The INCLUDE instruction copies the instructions from the specified member into the Jol input stream, thus replacing the INCLUDE statement.  The statement(s) that replaced the INCLUDE statement is then processed as though it was an original part of the input stream.

Applications of this facility include the storage of complete production job procedures and the storage of common procedures such as those used to invoke utility programs. Any instructions or card files that are liable to frequent change need not be stored in the member.  Instead, you can code declarations before the INCLUDE and place any additional instructions after it.  Instructions inside the INCLUDEd member may freely refer to definitions made outside of it, and later instructions may refer to definitions made inside the INCLUDEed member.

The INCLUDE member statement may be used to conveniently include a series of data set definitions.  This allows the installation to set up various members of the library with data set definitions, and anyone who wants to run part of that set of programs need only include the data set definition member, thus saving repetitive coding of his data sets.  Example three below explains this in more detail.

Card Image Data in INCLUDEd Text

 Data cards can be INCLUDEd in the text together with other text.  You can even replace parts (or all) of these data cards with the current values contained in Symbolic Variables.  See the Card File Declaration  for more information.

Notes
  1. The copied or INCLUDEd text is inserted immediately after the INCLUDE instruction.  Included Source Text may contain any Jol Macro Instructions or card image files.
  2. The INCLUDEd text may INCLUDE other members, up to a nesting level of twelve.  That is, an included member may include another member, and that member may include another member, and so on up to a level of twelve (12) concurrent INCLUDES.
  3. Card Image Data may be stored in text members for inclusion; these cards (perhaps utility control statements) may have variable data changed by using the symbolic parameter facility.
  4. The member name to be included may be defined by a symbolic parameter.  For example:
INCLUDE %DAY;

will INCLUDE member MONDAY or TUESDAY, etc.
  1. Member names must be eight characters or less, and must commence with an alphabetic or national ($, @ or #) character.
  2. Your installation may have several libraries concatenated to each other.  When you INCLUDE a member, each library is searched in the order in which they are concatenated.  This means that if two or more libraries contain identically named members, the first member found by the specified name will be the one to be included.
  3. After the instructions in the included member have been analyzed, the instructions following the INCLUDE are analyzed.
  4. Users with TSO can use the TSO EDIT Command or SPF to store and update members or procedures in the INCLUDE library.
  5. The files may ve Fixed or Variable Blocked files.

Examples
1.       INCLUDE SALES;
The INCLUDE instruction directs that the libraries are to be searched for the SALES member.  The text (Jol source statements) is then interpreted by Jol as though the contents of the INCLUDEd member SALES had been placed where the INCLUDE instruction had been.
After the instructions in SALES have been interpreted, the instruction following the INCLUDE will be interpreted.

2        IF %DAY = 'MONDAY'
THEN INCLUDE STARTWK;
Jol automatically initializes %DAY to the current day.  %DAY is tested; if it is MONDAY then the INCLUDE instruction is activated and the instructions in member STARTWK will be analyzed.

If the day is not MONDAY, the INCLUDE instruction will be ignored.

3       INCLUDE PAYSYS;
RUN VALIDATE;
LIST OUTPUT1;
The PAYSYS member is INCLUDEd.  In this example, PAYSYS contains all the program and data set definitions that a group of programmers are working on.  This programmer wishes to test a program called VALIDATE, and then list its output.  Instead of each programmer having separate procedures for their part of the system, all the common elements are coded and stored in member PAYSYS.  Whenever any programmer in the group wishes to test his or her program, he includes the common elements and then places his instructions after the INCLUDE.
In this example, the programmer includes the common elements, then RUN's the program VALIDATE and LISTs its output.
